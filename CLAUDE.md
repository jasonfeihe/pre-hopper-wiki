# Pre-Hopper Kernel Optimization Knowledge Base — Schema

A structured knowledge base of GPU kernel optimization for pre-Hopper NVIDIA
GPUs — Turing (`sm75`, T4), Ampere (`sm86`, A10), Ada Lovelace (`sm89`,
L20/L40) — optimized for LLM-agent retrieval. Hopper (`sm90`), Blackwell
(`sm100`), and A100 (`sm80`) are out of scope.

This file is the extended schema + navigation reference for Claude Code. The
condensed companion is `references/schema.md`; the skill entry point is
`SKILL.md`.

## Environment

`uv` + Python 3.12, single runtime dependency PyYAML. Always run the scripts
through uv:

```bash
uv sync
uv run python scripts/validate.py
uv run python scripts/generate-indices.py
```

## Navigation

```bash
uv run python scripts/query.py "<keywords>" [--type X --tag Y --architecture Z]
uv run python scripts/get_page.py <id-or-path> [--follow-sources]
uv run python scripts/grep_wiki.py "<pattern>" [--only wiki]
```

Pre-built indices under `queries/` (auto-generated; never edit by hand):
`by-problem.md`, `by-technique.md`, `by-hardware-feature.md`,
`by-kernel-type.md`, `by-language.md`, `by-repo.md`.

## Three-Layer Architecture

- **Layer 1 — `sources/`**: raw data with YAML frontmatter and a unique `id`.
  `sources/prs/{repo}/PR-{N}.md`, `sources/docs/*.md`, `sources/blogs/*.md`,
  `sources/contests/*.md`.
- **Layer 2 — `wiki/`**: synthesized pages, cross-referenced by `id`.
  `wiki/hardware/`, `wiki/techniques/`, `wiki/kernels/`, `wiki/patterns/`,
  `wiki/languages/`, `wiki/migration/`.
- **Layer 3 — `queries/`**: auto-generated indices. Regenerated by
  `scripts/generate-indices.py`.

## Page Types and Required Fields

See `data/schemas.yaml` for the authoritative definitions. Summary:

| Page Type | id prefix | Key Required Fields |
|-----------|-----------|---------------------|
| source-doc | `doc-` | title, url, source_category, architectures, tags, retrieved_at |
| source-blog | `blog-` | title, author, url, source_category, architectures, tags, retrieved_at |
| source-pr | `pr-` | repo, pr, title, author, date, url, source_category, architectures, tags, captured_at, status |
| source-contest | `contest-` | title, source_category, architectures, tags |
| wiki-hardware | `hw-` | title, type=hardware, architectures, tags, confidence, related, sources |
| wiki-technique | `technique-` | title, type=technique, architectures, tags, confidence, reproducibility(≥snippet), prerequisites, related, sources |
| wiki-pattern | `pattern-` | title, type=pattern, tags, symptoms, candidate_techniques, related, sources |
| wiki-kernel | `kernel-` | title, type=kernel, architectures, tags, confidence, reproducibility(≥snippet), kernel_types, languages, related, sources |
| wiki-language | `lang-` | title, type=language, tags, related, sources, reproducibility(≥snippet) |
| wiki-migration | `migration-` | title, type=migration, from_arch, to_arch, tags, related, sources |

`scope_relevance` is an **optional** free-text field available on every wiki page
to justify in-scope inclusion. There is **no** `blackwell_relevance` field and
**no** Blackwell-first validator rule (this KB is pre-Hopper-scoped).

## Controlled Vocabulary

All tags must appear in `data/tags.yaml`; unknown tags are rejected by the
validator.

- `architectures`: exactly `sm75`, `sm86`, `sm89`.
- `hardware_features`, `techniques`, `kernel_types`, `languages`,
  `confidence`, `reproducibility`, `source_categories`: see `data/tags.yaml`.

Aliases in `data/aliases.yaml` map device names and instruction spellings to
canonical tags:
- `sm75` = Turing = T4
- `sm86` = Ampere = A10
- `sm89` = Ada Lovelace = L20 = L40
- `mma-sync` = `mma.sync`; `cp-async` = `cp.async`; `fp8` = E4M3 = E5M2

## Confidence Rules

- `verified`: official-doc + upstream-code evidence
- `source-reported`: ≥ 1 authoritative source
- `inferred`: synthesized from multiple sources
- `experimental`: undocumented / version-sensitive

## Reproducibility Levels

`concept` < `pseudocode` < `snippet` < `runnable` < `benchmarked`. Minimum for
technique/kernel/language pages: `snippet`.

## Per-Architecture Feature Boundaries (authoring correctness)

- **Turing (sm75)**: `mma.sync`, `ldmatrix`, FP16/INT8/INT4 Tensor Cores; **no**
  `cp.async` (synchronous shared-memory staging only).
- **Ampere (sm86)**: adds `cp.async`, BF16, TF32.
- **Ada Lovelace (sm89)**: adds FP8 (E4M3/E5M2); retains `cp.async`.

Do not claim Ampere-only features (e.g. `cp.async`) on an sm75-only page.

## Tooling

- `scripts/validate.py` — schema, vocabulary, id, and link-integrity validation.
- `scripts/generate-indices.py` — deterministic regeneration of `queries/*.md`.
- `scripts/query.py` / `get_page.py` / `grep_wiki.py` — retrieval.
- `scripts/_wiki_root.py` — root resolver (honors `PREHOPPER_WIKI_ROOT`).

## Incremental Updates

The corpus is append-only and not frozen to a single point in time. See
`references/incremental-updates.md` for the add-a-page workflow and the
`data/refresh-cutoff.yaml` baseline semantics.

## Scope Rules

- **Pre-Hopper only**: `sm75`/`sm86`/`sm89`. No `sm80`/`sm90`/`sm100`.
- **English canonical**: all content in English.
- **Sourced**: every synthesized wiki page cites at least one resolvable source.
- **Design lineage**: original code inspired by KernelWiki; see `PROVENANCE.md`.
