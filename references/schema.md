# Pre-Hopper Kernel Wiki — Schema & Navigation Reference

A structured knowledge base of GPU kernel optimization for **pre-Hopper** NVIDIA
GPUs: Turing (`sm75`, T4), Ampere (`sm86`, A10), Ada Lovelace (`sm89`, L20/L40).
Optimized for LLM-agent retrieval. Scope excludes Hopper (`sm90`), Blackwell
(`sm100`), and A100 (`sm80`).

## Navigation

Run the skill scripts through `uv` from the repo root:

```bash
uv run python scripts/query.py "<keywords>" [--type X --tag Y --architecture Z]
uv run python scripts/get_page.py <id-or-path> [--follow-sources]
uv run python scripts/grep_wiki.py "<pattern>" [--only wiki]
```

Pre-built indices live under `queries/` (auto-generated; do not edit manually):
by-hardware-feature, by-technique, by-kernel-type, by-language, by-problem,
by-repo.

## Three-Layer Architecture

1. **`sources/`** — raw data. `sources/{docs,blogs,prs,contests}/`. Each file has
   YAML frontmatter with a unique `id`.
2. **`wiki/`** — synthesized knowledge pages, cross-referenced by `id`.
   `wiki/{hardware,techniques,kernels,patterns,languages,migration}/`.
3. **`queries/`** — auto-generated cross-reference indices. Regenerated by
   `scripts/generate-indices.py`; never edited by hand.

## Page Types and Required Fields

Defined in `data/schemas.yaml` and enforced by `scripts/validate.py`.

| Page type      | id prefix    | Key required fields |
|----------------|--------------|---------------------|
| source-doc     | `doc-`       | id, title, url, source_category, architectures, tags, retrieved_at |
| source-blog    | `blog-`      | id, title, author, url, source_category, architectures, tags, retrieved_at |
| source-pr      | `pr-`        | id, repo, pr, title, author, date, url, source_category, architectures, tags, captured_at, status |
| source-contest | `contest-`   | id, title, source_category, architectures, tags |
| wiki-hardware  | `hw-`        | id, title, type=hardware, architectures, tags, confidence, related, sources |
| wiki-technique | `technique-` | id, title, type=technique, architectures, tags, confidence, reproducibility(≥snippet), prerequisites, related, sources |
| wiki-pattern   | `pattern-`   | id, title, type=pattern, tags, symptoms, candidate_techniques, related, sources |
| wiki-kernel    | `kernel-`    | id, title, type=kernel, architectures, tags, confidence, reproducibility(≥snippet), kernel_types, languages, related, sources |
| wiki-language  | `lang-`      | id, title, type=language, tags, related, sources, reproducibility(≥snippet) |
| wiki-migration | `migration-` | id, title, type=migration, from_arch, to_arch, tags, related, sources |

Every wiki page may carry an **optional** `scope_relevance` free-text field
explaining why it belongs in the pre-Hopper scope. No page type requires a
`blackwell_relevance` field, and the validator enforces no Blackwell-first rule.

Every synthesized wiki page must cite at least one resolvable source in
`sources:`. `sources:` and `related:` entries must resolve to an existing page
`id` (link integrity is enforced).

## Controlled Vocabulary

All tags must appear in `data/tags.yaml`; unknown tags are rejected.

- **architectures**: exactly `sm75`, `sm86`, `sm89`.
- **hardware_features**: e.g. `tensor-core`, `mma-sync`, `wmma`, `ldmatrix`,
  `cp-async`, `async-copy`, `shared-memory`, `tf32`, `bf16`, `fp16`, `int8`,
  `int4`, `fp8`, `dp4a`, `warp-shuffle`, `async-barrier`. Blackwell-only features
  (`tcgen05`, `tmem`, `clc`, `nvfp4`, `2sm-cooperative`) are intentionally absent.
- **techniques**, **kernel_types**, **languages**: see `data/tags.yaml`.
- **confidence**: `verified` > `source-reported` > `inferred` > `experimental`.
- **reproducibility**: `concept` < `pseudocode` < `snippet` < `runnable` <
  `benchmarked`. Minimum for technique/kernel/language pages is `snippet`.

### Aliases (data/aliases.yaml)

Device/marketing names and instruction spellings map to canonical tags, e.g.:

- `T4`, `Turing` → `sm75`
- `A10`, `Ampere` → `sm86`
- `L20`, `L40`, `Ada`, `Ada Lovelace` → `sm89`
- `mma.sync` → `mma-sync`; `cp.async` → `cp-async`; `FP8`/`E4M3`/`E5M2` → `fp8`

`query.py` expands aliases for `--architecture` and `--tag` filters and for
keyword scoring.

## Per-Architecture Feature Notes

These pairings are correctness-relevant when authoring pages:

- **Turing (sm75)**: `mma.sync`, `ldmatrix`, FP16/INT8/INT4 Tensor Cores. **No
  `cp.async`** — shared-memory staging is synchronous (register round-trip).
- **Ampere (sm86)**: adds `cp.async` asynchronous copy, BF16, TF32.
- **Ada Lovelace (sm89)**: adds FP8 (E4M3/E5M2) on top of the Ampere set;
  retains `cp.async`.

## Tooling

- `scripts/validate.py` — validates frontmatter against `data/schemas.yaml`, the
  controlled vocabulary, id uniqueness/prefixes, and `sources`/`related` link
  integrity. Exit 0 = clean.
- `scripts/generate-indices.py` — regenerates `queries/*.md` deterministically.
- `scripts/query.py`, `scripts/get_page.py`, `scripts/grep_wiki.py` — retrieval.
- `scripts/_wiki_root.py` — resolves the knowledge-base root (honors
  `PREHOPPER_WIKI_ROOT`, else autodetects).

Setup: `uv sync`. Run: `uv run python scripts/validate.py` then
`uv run python scripts/generate-indices.py`.
